# SpatialScope and QuPath: Functional Comparison

**Last updated:** July 18, 2026<br>
**SpatialScope reference:** SpatialScope application behavior and standard generated outputs<br>
**QuPath reference:** QuPath 0.7.0 official documentation

## Scope of this comparison

SpatialScope and QuPath overlap in multiplex image analysis, cell detection, phenotyping, region analysis, and spatial measurements, but they are designed around different starting points. QuPath is a broad, cross-platform bioimage-analysis environment with particular strength in digital pathology, whole-slide images, interactive annotation, machine learning, scripting, and format support. SpatialScope is a focused macOS workflow that takes per-marker numeric intensity matrices through a fixed sequence of tumor-microenvironment analyses and produces analysis-ready and publication-ready outputs.

This is a functional and workflow comparison, not an accuracy or speed benchmark. Because QuPath is open source, scriptable, and extensible, it would be misleading to describe most calculations as impossible in QuPath. In the table, **"not native" means that the standard QuPath 0.7.0 interface and official documentation do not provide an equivalent first-class, end-to-end workflow**. A QuPath user could potentially reproduce such behavior with input conversion, Groovy scripts, an extension, or external statistical software.

| Distance analysis and statistics | Computes target-to-query nearest-neighbor distances and cell-to-boundary distances with inside, outside, or all-cell filters. | Reproducibility and automation | Saves pipeline configuration, stage parameters, screening records, analysis manifests, output inventories, and CPU/resource metadata while showing stage readiness and completion in the interface. | Records many commands and parameters in command history and workflows; workflows can be converted to scripts for batch processing across projects.[11][12] | Both support reproducibility. ## SpatialScope-specific tasks not available as native QuPath workflows

1. **Directly reconstruct and analyze a multiplex image from a folder of per-marker numeric CSV matrices.** The user does not need to convert those matrices into OME-TIFF or another QuPath-readable image first.
2. **Run adaptive, ranked parameter screening for both nuclei segmentation and rule-based cell-type assignment.** SpatialScope preserves the tested combinations, recommended combination, figures, and metadata.
3. **Create fixed-area neighborhood identities from exact cell-type co-composition.** SpatialScope generates a tile-level composition key, cluster mask, key table, map, and summary together.
4. **Generate calibrated inside/outside boundary bands and immediately quantify cell density and neighborhood-cluster distribution against those bands or regions.** The linked band maps, long/wide tables, region summaries, NPZ arrays, plots, and heatmaps are produced as one workflow.
5. **Combine distance calculations with built-in group statistics and publication figures.** QuPath calculates useful spatial measurements, while SpatialScope directly turns selected comparisons into statistical tables and coordinated PNG, Illustrator-compatible AI/PDF, and SVG outputs.
6. **Produce a complete analysis-stage export packet automatically.** SpatialScope saves figures, masks, cell-level data, parameters, and manifests in a predictable directory structure without requiring a separate export script.

## Short assessment

SpatialScope's main advantage is not that it replaces every capability in QuPath. Its advantage is **workflow specificity**. For a study that begins with aligned per-marker CSV intensity matrices and needs rule-based immune-cell phenotyping, cell-type-driven ROIs, compositional neighborhoods, boundary gradients, distance statistics, and editable publication figures, SpatialScope reduces format conversion, scripting, external statistics, and manual export assembly. The entire analysis follows a visible dependency order, and every stage produces outputs intended for both inspection and downstream computation.

QuPath remains highly complementary. It is especially valuable for viewing and annotating large pyramidal slides, working across many vendor image formats, training interactive classifiers, using deep-learning extensions, and automating diverse projects through Groovy. A productive positioning is therefore: **QuPath is a broad digital-pathology platform, while SpatialScope is a specialized, end-to-end TME spatial-analysis application for matrix-based multiplex data.**
